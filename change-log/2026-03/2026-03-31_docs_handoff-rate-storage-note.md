# 운영 인수인계 문서에 환율/스토리지 운영 메모 보강

## 요약
- 운영 인수인계 문서에 환율 갱신이 외부 운영자 서버의 별도 시스템에 의존한다는 점을 명시했다.
- 현재 S3 호환 오브젝트 스토리지가 iwinv 기반이라는 점과, 이관 시 기존 정적 이미지는 유지하고 신규 스토리지를 단계 전환해야 할 수 있다는 점을 추가했다.

## 상세 변경
- `운영인수인계.md`
  - 환율 업데이트 섹션에 운영자 자체 서버에서 외부 크론/웹훅 시스템을 구축해 사용 중이라는 설명 추가
  - S3 호환 오브젝트 스토리지 섹션에 iwinv 사용 사실과 스토리지 이관 전략 메모 추가
  - 운영 리스크 메모에 외부 환율 시스템 인수인계와 스토리지 단계 전환 필요성을 보강
- `todo.md`
  - 이번 문서 보강 작업 항목을 완료 처리로 기록

## 운영 메모
- 환율 시스템은 Render 내부만 보면 전체 운영 흐름이 보이지 않으므로, 외부 서버 접근 권한과 배치 설정까지 함께 인계해야 한다.
- 오브젝트 스토리지 이관 시 기존 파일 URL 호환성을 우선 보장하고 신규 업로드부터 전환하는 전략이 안전하다.
