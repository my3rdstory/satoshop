# 개발 서버 로컬 IP 접근 허용

## 변경 내용
- `DEBUG=True`인 개발 환경에서 `ALLOWED_HOSTS`를 넓게 허용하도록 조정했다.
- `runserver 0.0.0.0:8011`로 실행한 뒤 동일 네트워크의 다른 기기에서 로컬 IP로 접속할 때 발생하던 `400 Bad Request`를 제거했다.
- README에 로컬 네트워크 접속 방법과 동작 방식을 문서화했다.

## 영향 범위
- 개발 서버 접근성
- Django 호스트 허용 설정

## 검증 메모
- 서버를 `uv run python manage.py runserver 0.0.0.0:8011`로 실행한다.
- 같은 장비에서는 `http://localhost:8011`로 접속한다.
- 같은 네트워크의 다른 기기에서는 `http://현재장비의로컬IP:8011`로 접속해 첫 화면이 렌더되는지 확인한다.
