## 배경
- 브랜드 홍보용 미니홈 랜딩을 슬러그 기반으로 운영하고, 주인장이 직접 섹션을 구성할 수 있는 관리 화면이 필요했음.

## 변경 사항
- `minihome` 앱을 추가하고 슬러그/주인장/드래프트/공개 섹션을 관리하는 모델과 어드민을 구성.
- `/minihome/<slug>/mng/`에서 섹션 추가 → 미리보기 → 공개하기 흐름으로 랜딩을 구성하도록 UI/JS를 구현.
- 브랜드/갤러리/미니블로그/CTA 이미지 업로드 시 WebP 변환 및 리사이즈 후 S3에 업로드하도록 처리.
- 미니홈 랜딩에서 네이버 지도 API 기반 지도 섹션과 미니 블로그 모달 미리보기 UI를 추가.

## 영향
- 관리자에서 지정한 주인장만 미니홈 관리 화면에 접근 가능하며, 공개 시 `/minihome/<slug>/`에 랜딩이 반영됨.

## 대장 수동 테스트 가이드
1. Django Admin에서 미니홈을 생성하고 슬러그와 주인장(복수)을 지정.
2. `/minihome/<slug>/mng/`에서 섹션을 추가하고 미리보기 버튼으로 새 탭 확인.
3. 공개하기 버튼으로 랜딩 페이지(`/minihome/<slug>/`) 반영 여부 확인.
4. 브랜드/갤러리/미니블로그/CTA 이미지 업로드 시 WebP 변환 및 리사이즈 적용 여부 확인.
5. 지도 섹션에서 위도/경도 입력 후 네이버 지도 표시 여부 확인.
