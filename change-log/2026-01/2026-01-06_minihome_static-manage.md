## 배경
- 미니홈 공개 페이지를 정적화하고, 갤러리/미니블로그/매장 편집을 다시 mng에서 처리해야 했음.

## 변경 사항
- mng에서 갤러리/미니블로그/매장 추가·수정·삭제를 직접 편집하도록 UI와 수집 로직을 재구성.
- mng 편집 영역을 랜딩 레이아웃과 동일한 카드 레이아웃으로 정렬.
- 공개 시 미니홈 HTML을 정적 파일로 생성하고, 공개 랜딩은 DB 없이 정적 파일을 서빙하도록 전환.
- 도메인 매핑은 정적 공개 맵 파일 기반으로 조회하도록 변경.

## 영향
- 공개된 미니홈은 DB 조회 없이 정적 파일로 제공됨.
- 갤러리/블로그/매장 편집은 mng에서만 처리됨.

## 대장 수동 테스트 가이드
1. `/minihome/<slug>/mng/`에서 갤러리/블로그/매장 아이템을 추가·수정·삭제.
2. 임시저장 후 페이지 리로드 시 편집 내용이 유지되는지 확인.
3. 공개하기 클릭 후 `/minihome/<slug>/`에서 정적 페이지가 반영되는지 확인.
4. 도메인 매핑이 설정된 경우 해당 도메인 루트에서 동일한 정적 페이지가 노출되는지 확인.
