# 스토어 상품 주문 송장 이메일 발송 기능

## 주요 변경
- 스토어 주인장이 상품별 주문 목록 화면에서 바로 송장 안내 메일을 발송할 수 있도록 `send_tracking_email_to_buyer` 서비스와 전용 API 엔드포인트를 추가했습니다.
- 송장번호 입력란 옆에 이메일 발송 열을 신설하고, 조건 불충족 시 사유를 안내하는 버튼/알림 UI와 Ajax 로직을 연동했습니다.
- 발송 버튼 하단에 실시간 상태 메시지를 표기해 전송 중/성공/실패 여부와 시간을 즉시 확인할 수 있도록 개선했습니다.
- 송장번호가 비어있거나 스토어 이메일 설정이 비활성화된 경우 버튼이 자동으로 비활성화되고, 조건을 충족해야 한다는 안내 문구가 즉시 표시됩니다.
- 스토어 이메일 설정이 비활성화되었을 때의 경고 배너와 README 주문 관리 섹션을 업데이트해 기능을 문서화했습니다.

## 확인 방법
1. 스토어 설정에서 이메일 발송 기능과 Gmail 계정을 활성화합니다.
2. 상품 주문 목록(`/orders/<store_id>/products/<product_id>/orders/`)에서 송장번호/택배사를 입력하고 저장합니다.
3. “이메일 발송” 버튼을 눌러 전송 완료 상태를 확인하고, 수신자 메일함에서 송장 안내 메일을 확인합니다.
