## 개요
- 직접 계약 검토/초대 화면에 결제 정책 위젯을 추가해 장고 어드민의 사토시 요금 정책을 이행해야 서명 버튼이 활성화되도록 막았습니다.
- Blink API로 60초짜리 인보이스를 발행/취소/상태 조회하는 HTMX + JS 흐름을 구성하고, 만료/오류 시 섹션만 새로고침해 재시도할 수 있게 했습니다.
- `EXPERT_BLINK_*` 환경 변수 및 README 가이드를 추가해 운영자가 손쉽게 월렛 정보를 설정할 수 있도록 정리했습니다.
- 결제 성공 내역을 `DirectContractDocument.payment_meta`·`DirectContractStageLog.meta`에 기록해 새로고침 후에도 결제 완료 상태를 기억하고, 어드민 1단계 목록에서는 시작 시간 역순 + 단계별 ⚡ 배지를 통해 결제 여부/시간/해시를 한눈에 확인하도록 했습니다.
- 검토 화면과 상대방 서명 화면 모두에 개인정보/비밀 유지/중계 책임/계약 확인 체크를 동일하게 노출해, 어느 쪽에서도 같은 조항을 확인하고 서명하도록 정비했습니다.

## 비고
- 테스트는 수동으로 Blink 자격 정보를 설정한 뒤 리뷰/초대 화면에서 결제 → 서명을 순차적으로 진행해 확인해 주세요.
