## 배경
- 디스코드 슬래시 명령어가 동일 이름으로 2번씩 보이는 문제가 발생했습니다.
- 원인은 글로벌 명령어와 길드 명령어가 동시에 등록된 상태였습니다.

## 변경 사항
- `sync_discord_commands` 기본 동작을 `길드 전용 전체 동기화`로 변경했습니다.
- 길드 동기화 시 기본적으로 기존 글로벌 명령어를 빈 목록으로 정리해 중복 노출을 방지하도록 개선했습니다.
- 필요 시 글로벌 유지가 가능하도록 `--keep-global` 옵션을 추가했습니다.
- `DiscordBot` 저장 시 자동 동기화 시그널도 동일하게 `길드 전용 + 글로벌 정리` 정책으로 통일했습니다.

## 영향
- 디스코드 입력창에서 `/사토샵_최근등록상품`, `/사토샵_최근판매상품`이 중복 없이 단일 항목으로 노출됩니다.
- 길드 명령어는 즉시 반영되며, 응답은 기존대로 에페메랄(실행자 본인만 보기)로 유지됩니다.

## 대장 수동 테스트 가이드
1. `uv run python manage.py sync_discord_commands --all-guilds` 실행.
2. 디스코드에서 `/사토샵_최근등록상품`, `/사토샵_최근판매상품` 자동완성이 각각 1개씩만 보이는지 확인.
3. 명령 실행 결과가 실행자 본인에게만 보이는지 확인.
