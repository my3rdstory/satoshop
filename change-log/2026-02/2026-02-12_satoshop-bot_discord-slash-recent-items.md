## 배경
- 디스코드에서 사토샵 전체 아이템의 최근 등록/최근 판매 항목을 빠르게 조회할 수 있는 슬래시 명령어가 필요했습니다.
- 최근 판매 목록은 다중 판매 항목에 대한 누적 건수 표시가 필요했습니다.

## 변경 사항
- `satoshop_bot`에 디스코드 인터랙션 엔드포인트(`/satoshop-bot/discord/interactions/`)를 추가하고 Ed25519 서명 검증 로직을 적용했습니다.
- 슬래시 명령어 2종(`/사토샵_최근등록상품`, `/사토샵_최근판매상품`) 처리 및 선택형 목록 응답을 구현했습니다.
- 최근 등록/최근 판매 통합 집계 서비스를 추가해 상품·밋업·라이브강의·디지털파일을 최근순으로 20개 반환하도록 구현했습니다.
- 가격 표시는 사토시 기준으로 통일했고, 원화 연동 아이템은 기존 모델의 환율 반영 사토시 값을 사용했습니다.
- 최근 판매 목록은 동일 아이템의 판매 건수를 집계해 `(판매: N건)` 포맷으로 노출되도록 구현했습니다.
- 목록 웹뷰 페이지 2종을 추가해 디스코드에서 링크 버튼으로 최근 목록을 바로 열 수 있게 했습니다.
- 디스코드 명령어 등록 자동화를 위해 `uv run python manage.py sync_discord_commands` 관리 명령을 추가했습니다.
- 디스코드 봇 설정 모델에 `application_id`, `public_key` 필드를 추가했습니다.

## 영향
- 디스코드에서 사토샵 최근 등록/판매 아이템 탐색과 상세 진입이 가능해졌습니다.
- 최근 판매 데이터는 판매 시점 기준 정렬과 누적 판매건수 표시를 함께 제공합니다.

## 대장 수동 테스트 가이드
1. Django Admin > 봇 연동 > 디스코드 봇에서 `token`, `application_id`, `public_key`를 설정하고 활성화합니다.
2. `uv run python manage.py sync_discord_commands` 실행 후 명령어 2개가 등록되었는지 확인합니다.
3. 디스코드에서 `/사토샵_최근등록상품` 실행 시 최근 등록 20개가 `아이템명 / 가격` 형식으로 노출되는지 확인합니다.
4. 디스코드에서 `/사토샵_최근판매상품` 실행 시 최근 판매 20개가 노출되고 다중 판매 항목은 `(판매: N건)`이 함께 표시되는지 확인합니다.
5. 목록에서 항목 선택 시 상세 링크 버튼이 노출되고 사토샵 상세 페이지로 이동되는지 확인합니다.
6. 목록 웹뷰 버튼이 `/satoshop-bot/webview/recent-registered/`, `/satoshop-bot/webview/recent-sold/` 페이지로 정상 이동되는지 확인합니다.
