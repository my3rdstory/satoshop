## 배경
- 개발 서버 테스트 채널에서 슬래시 명령어가 바로 동작하지 않는 문제가 있었고, 서버 전체 채널에서 즉시 사용 가능한 동기화 방식이 필요했습니다.
- 명령어 응답은 실행자 본인에게만 보이도록 유지해야 했습니다.

## 변경 사항
- `sync_discord_commands`를 글로벌 단일 동기화에서 `글로벌 + 길드 동기화` 지원으로 확장했습니다.
- 기본 실행(`uv run python manage.py sync_discord_commands`) 시 봇이 가입한 모든 길드 ID를 조회해 길드 명령어까지 함께 동기화하도록 변경했습니다.
- 길드 동기화 제어 옵션을 추가했습니다.
  - `--all-guilds`
  - `--guild-id <ID>` (복수 지정 가능)
  - `--guild-only`
  - `--global-only`
- 인터랙션 응답은 기존대로 에페메랄(`flags=64`)로 유지되어 실행자 본인에게만 노출됩니다.

## 영향
- 테스트/운영 서버에서 길드 명령어를 즉시 반영해 채널 전반에서 빠르게 사용 가능합니다.
- 글로벌 명령어 전파 지연이 있어도 길드 명령어를 통해 우선 동작 확인이 가능합니다.

## 대장 수동 테스트 가이드
1. `uv run python manage.py sync_discord_commands --all-guilds` 실행.
2. 테스트 서버 아무 채널에서 `/사토샵_최근등록상품` 또는 `/사토샵_최근판매상품` 실행.
3. 다른 유저 화면에는 보이지 않고 실행자 본인에게만 응답이 보이는지 확인.
4. 즉시 반영 확인이 필요하면 `--guild-only --all-guilds`로 재실행 후 다시 테스트.
