## ADDED Requirements

### Requirement: Expert Gmail 발송 정보 관리
- Expert 계약 이메일 발송에 필요한 Gmail 주소/앱 비밀번호/발신자명은 장고 어드민 `expert/expertemailsettings/` 경로에서 등록/수정할 수 있어야 하며, 해당 화면은 이를 **MUST** 지원한다.
- Gmail 설정이 하나도 없을 때도 관리자가 "추가" 버튼을 눌러 최초 레코드를 생성할 수 있어야 하며, 이후에는 단일 레코드만 유지한다.

#### Scenario: Gmail 설정 최초 등록
- **GIVEN** Expert 이메일 설정 목록에 레코드가 없을 때
- **WHEN** 관리자가 "추가"를 눌러 Gmail 주소와 앱 비밀번호를 저장하면
- **THEN** 단일 설정 레코드가 생성되고 목록/수정 화면에서 다시 편집할 수 있어야 한다.

### Requirement: 분할 지급 세부 입력
- 계약 드래프트에서 분할 지급을 선택하면 각 회차의 금액, 지급 예정일, 지급 조건 메모를 입력할 수 있는 필드를 **MUST** 제공하고, 총 금액 대비 합계 검증을 통과해야 한다.
- 입력된 분할 지급 정보는 검토/초대/미리보기 화면과 최종 PDF 요약에 회차별로 표시되어야 한다.

#### Scenario: 분할 지급 세부 확인
- **GIVEN** 사용자가 분할 지급을 선택하고 두 개 이상의 회차를 작성했을 때
- **WHEN** 계약 미리보기 또는 검토 화면을 열면
- **THEN** 회차 번호별로 금액, 지급일(미입력 시 "미정"), 지급 조건 메모가 모두 노출되고, 총액 초과 입력은 거부된다.

### Requirement: 수행 내역 메모 노출
- 사용자가 작성한 수행 내역 메모(Markdown)는 계약서 미리보기의 "2. 계약 내용" 블록과 검토/초대 화면에서 그대로 렌더링되어야 하며, 모든 화면은 이를 **MUST** 반영한다.

#### Scenario: 미리보기에서 메모 반영
- **GIVEN** 사용자가 수행 내역 메모를 작성했을 때
- **WHEN** 계약서 미리보기를 열면
- **THEN** "2. 계약 내용" 영역 하단에 전체 메모 내용이 Markdown 스타일로 변환되어 표시된다.

### Requirement: PDF 전용 첨부 및 출력
- 계약 첨부는 서버 측에서도 PDF 확장자인지 검증하여 비-PDF 항목은 MUST 거부하거나 제외해야 한다.
- 업로드된 PDF 목록은 계약 미리보기/검토/초대 화면 하단과 최종 PDF 마지막 섹션에 붙임 형태로 나열되어야 하며(파일명과 링크/경로 포함), 각 화면은 이를 MUST 출력한다.

#### Scenario: 첨부 PDF 표시
- **GIVEN** 사용자가 PDF를 첨부했을 때
- **WHEN** 계약 미리보기, 검토 화면 또는 최종 PDF를 확인하면
- **THEN** 첨부 섹션에서 각 파일명이 PDF로 표기되고, 다운로드 링크(웹 화면) 또는 경로 정보(PDF)가 제공된다.
